# 사용자와 상황

## 사용자

### 여행 멤버

- 인원: 친구 네 명
- 기기: 대부분 모바일 브라우저
- 상황: 이동 중, 식당 대기 중, 숙소에서 다음 날 준비 중
- 기대: 복잡한 조작 없이 오늘 일정과 후보 장소를 바로 확인한다.

### 관리자

- 관리자는 주환이다.
- 주환은 관리자 접근 코드 입력 후 관리자 권한을 사용한다.
- 기본 여행 계획을 작성하고 관리한다.
- 친구들이 올린 후보 장소를 승인, 보류, 거절한다.
- 승인한 장소를 특정 날짜나 일정에 연결한다.
- 실제 방문한 장소를 방문 완료로 표시한다.

### 장소 제안자

- 식당, 카페, 디저트, 관광 후보를 추가한다.
- 구글 지도 링크와 추천 이유를 함께 남긴다.
- 다른 친구들의 추천과 코멘트를 확인한다.

## 주요 사용 시나리오

### 여행 전

- 관리자가 요일별 기본 일정을 입력한다.
- 네 명이 각자 자기 이름을 선택해 접속한다.
- 친구들이 가고 싶은 식당, 카페, 관광 후보를 구글 지도 링크로 추가한다.
- 후보 장소에 추천과 코멘트를 남긴다.
- 관리자가 의견을 보고 최종 후보를 승인한다.

### 여행 중 아침

- 오늘 일정 전체를 확인한다.
- 다음 이동 장소, 출발 시간, 지도 링크를 연다.
- 오늘 승인된 식당/카페/후보 장소를 확인한다.

### 이동 중

- 다음 일정의 주소와 지도 링크를 바로 연다.
- 근처 후보 장소를 확인한다.
- 새로 발견한 장소를 후보로 추가한다.
- 일정과 연결된 승인 장소를 확인한다.

### 장소를 고를 때

- 후보 목록을 추천순으로 본다.
- 코멘트에서 웨이팅, 예약, 가격대 같은 메모를 확인한다.
- 관리자가 최종 방문지를 승인한다.

### 방문 후

- 주환이 방문한 장소를 방문 완료로 표시한다.
- 멤버들은 방문 완료 상태를 확인한다.
- 다음 여행을 위해 좋았던 장소를 남겨둔다.
