# 기능 명세

## 1. 홈

### 목적

오늘의 기본 일정과 승인된 방문 후보를 첫 화면에서 빠르게 확인한다.

### 포함 정보

- 오늘 날짜와 도시
- 현재 선택된 멤버
- 오늘 일정 요약
- 다음 일정 카드
- 오늘 승인된 방문 후보
- 빠른 메뉴: 일정, 후보 추가, 회계, 승인 대기

### 우선순위

MVP 필수.

## 2. 이름 선택

### 목적

네 명이 각자 자기 이름으로 들어와 추천, 코멘트, 방문 확인을 남긴다.

### 기능

- 첫 접속 시 멤버 이름 선택
- 선택 가능한 멤버: 주환, 지환, 지완, 지원
- 각 멤버는 고유 아이디와 테마 색상을 가짐
- 선택한 이름은 브라우저에 저장
- 관리자는 주환
- 주환은 관리자 접근 코드 입력 후 관리자 권한 사용
- 관리자 접근 코드는 환경변수 `ADMIN_ACCESS_CODE`로 관리
- 지환, 지완, 지원은 이름 선택만으로 입장
- 현재 선택된 이름 표시
- 이름 변경 가능
- 내가 남긴 후보, 추천, 댓글, 방문 확인은 내 이름과 색상으로 표시

### 우선순위

MVP 필수.

## 3. 기본 일정

### 목적

관리자가 짜둔 날짜/요일별 여행 계획을 모두가 같은 구조로 본다.

### 기능

- 날짜별 일정 보기
- 기본 일정은 관리자가 짠 고정 동선으로 표시
- 일반 멤버는 기본 일정 자체를 수정할 수 없음
- 요일, 도시, 숙소/이동/관광 같은 큰 흐름 표시
- 시간대, 제목, 설명, 장소명, 구글 지도 링크 표시
- 날짜별 주요 장소 구글맵 링크 표시
- 장소명 또는 지도 버튼을 누르면 구글맵으로 열기
- 예약/이동처럼 중요한 일정 강조
- 일정 항목 사이에 관련 후보 장소 연결
- 일정 화면 안에서 후보 추가, 추천, 코멘트, 방문 확인 가능
- 관리자라면 일정 화면 안에서 후보 승인 가능
- 관리자는 기본 일정 수정 가능

### 우선순위

MVP 최우선.

## 4. 후보 장소

### 목적

친구들이 기본 동선 사이에 넣고 싶은 식당, 카페, 관광 후보를 구글 지도 링크로 추가한다.

### 기능

- 후보 장소 추가
- 후보 장소는 별도 하단 탭이 아니라 일정 화면 안에서 추가
- 구글 지도 링크를 붙여넣으면 후보 장소로 등록
- 필수 입력: 장소명, 도시, 카테고리, 구글 지도 링크
- 선택 입력: 추천 이유, 관련 날짜, 관련 일정
- 후보를 추가한 위치는 자동으로 "이 기본 일정 사이"로 저장
- 카테고리: 식당, 카페, 디저트, 바, 관광, 쇼핑, 기타
- 상태: 제안됨, 승인됨, 보류, 거절, 방문 완료
- 제안자 표시
- 제안자 이름과 테마 색상 표시

### 우선순위

MVP 필수.

## 5. 추천

### 목적

후보 장소에 대한 선호도를 빠르게 모은다.

### 기능

- 멤버별 추천 토글
- 추천은 네 명 모두 가능
- 추천 수 표시
- 내가 추천한 장소 표시
- 추천한 멤버 목록 표시
- 추천한 멤버의 이름과 테마 색상 표시
- 추천순 정렬

### 우선순위

MVP 필수.

## 6. 코멘트

### 목적

후보 장소에 대한 의견, 예약 필요 여부, 웨이팅 정보 등을 남긴다.

### 기능

- 장소별 코멘트 작성
- 작성자와 작성 시간 표시
- 작성자 이름과 테마 색상 표시
- 코멘트 삭제는 작성자 또는 관리자만 가능
- 짧은 메모 중심의 모바일 입력

### 우선순위

MVP 필수.

## 7. 관리자 승인

### 목적

관리자가 최종 방문 후보를 확정하고 기본 동선 사이에 배치한다.

### 기능

- 승인 대기 장소 목록
- 승인 관리는 별도 하단 탭이 아니라 일정 화면 또는 더보기에서 진입
- 승인, 보류, 거절 처리
- 승인 시 후보가 추가된 기본 일정 사이에 확정 후보로 표시
- 승인 사유 또는 메모
- 승인된 장소만 일정 화면의 확정 후보로 표시
- 승인되어도 기존 기본 일정은 삭제하거나 대체하지 않음

### 우선순위

MVP 필수.

## 8. 방문 확인

### 목적

실제로 방문한 장소를 기록한다.

### 기능

- 승인된 장소를 방문 완료로 표시
- 방문 완료는 관리자만 처리 가능
- 방문한 멤버 선택
- 방문 확인을 남긴 멤버 표시
- 방문한 멤버 이름과 테마 색상 표시
- 짧은 후기 또는 메모
- 방문 완료 장소 목록

### 우선순위

MVP 필수.

## 9. 회계

### 목적

개인 지출과 공동 지출을 구분해서 기록하고, 공동 지출은 앱에서 자동으로 정산한다.

### 비용 종류

- **나에게 쓴 돈**: 개인 기록용. 정산 계산에는 포함하지 않는다.
- **모두에게 쓴 돈**: 네 명 공동 비용. 앱에서 1/N 정산 계산에 포함한다.

### 기능

- 비용 항목 추가
- 비용 종류 선택: 나에게 쓴 돈, 모두에게 쓴 돈
- 결제자 선택
- 금액, 통화, 날짜, 카테고리, 메모 입력
- 통화는 EUR, KRW 지원
- 환율 자동 계산으로 EUR/KRW 환산 금액 표시
- 공동 비용은 기본적으로 네 명 전체 1/N
- 공동 비용 정산 요약 표시
- 누가 누구에게 얼마를 보내야 하는지 표시
- 개인별 총 결제액, 개인 부담액, 받을 금액, 보낼 금액 표시
- 정산 완료 버튼
- 네 명 모두 정산 완료를 눌러야 공동 비용 정산 라운드가 초기화됨
- 정산 완료 후 기존 공동 비용은 정산 완료 기록으로 보관하고, 새 공동 비용 계산은 0부터 시작
- 나에게 쓴 돈은 정산 초기화와 무관하게 개인 기록으로 유지

### 정산 규칙

- 공동 비용 합계를 네 명이 동일하게 나눈다.
- 서로 다른 통화가 섞이면 자동 환율로 기준 통화 금액을 계산한다.
- 많이 낸 사람은 받을 사람, 적게 낸 사람은 보낼 사람이 된다.
- 앱은 최소 송금 횟수에 가깝게 "A가 B에게 얼마" 형태로 정산안을 만든다.
- 네 명 모두가 정산 완료를 눌러야 현재 정산 라운드를 닫는다.

### 우선순위

MVP 필수.

## 11. 설정

### 목적

여행 기본값과 멤버 정보를 관리한다.

### 기능

- 여행 이름
- 여행 기간
- 도시 목록
- 멤버 이름
- 멤버 아이디와 테마 색상
- 관리자 멤버
- 공유 링크 또는 간단한 접근 코드
- 환율 API 설정

### 우선순위
