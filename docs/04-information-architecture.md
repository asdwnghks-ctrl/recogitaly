# 정보 구조와 화면

## 내비게이션

모바일 하단 탭을 기본으로 한다.

- 홈
- 일정
- 회계
- 더보기

후보 장소는 일정 안에서 날짜/일정 사이에 붙는 레이어로 다룬다. 관리자가 아닌 멤버에게는 승인 관리 메뉴를 숨기거나 읽기 전용으로 둔다. 더보기 안에는 멤버 변경, 설정, 정산 기록을 둔다.

## 화면 구조

### 진입

- 앱 타이틀: 이탈리아 가아즈아
- 멤버 이름 선택
- 관리자 접근 코드 입력
- 선택한 이름 저장

### 홈

- 상단: 앱 타이틀, 현재 도시, 날짜
- 현재 선택된 멤버
- 오늘 일정 요약
- 다음 일정
- 오늘 승인된 후보 장소
- 빠른 링크

### 일정

- 날짜 선택 탭
- 타임라인 목록
- 일정 상세
- 날짜별 후보 장소
- 일정 항목 사이의 후보 장소
- 승인된 후보와 제안된 후보 구분
- 후보 추가 버튼
- 추천, 코멘트, 구글 지도 열기
- 방문 완료 표시
- 관리자용 일정 수정 버튼
- 관리자용 승인 관리

### 일정 안의 후보

- 날짜 필터
- 카테고리 필터
- 상태 필터: 제안됨, 승인됨, 보류, 거절, 방문 완료
- 후보 장소 카드 목록
- 추천 버튼
- 코멘트
- 구글 지도 열기
- 후보 추가 버튼

### 일정 안의 승인 관리

- 관리자 전용 화면
- 승인 대기 후보 목록
- 추천 수와 코멘트 요약
- 승인/보류/거절 액션
- 승인 시 연결 날짜와 일정 선택

### 회계

- 개인 비용과 공동 비용 입력
- 현재 정산 라운드 요약
- 누가 누구에게 얼마를 줄지 표시
- 정산 완료 버튼
- 정산 완료 상태

### 더보기

- 멤버 변경
- 여행 설정
- 정산 기록

## UI 원칙

- 모바일 기준으로 먼저 설계한다.
- 전체 톤은 귀엽고 장난기 있는 단톡방 여행앱으로 잡는다.
- 배경은 흰색을 기본으로 하고, 정보 카드에는 아주 연한 회색 표면을 사용한다.
- 앱 타이틀은 `Bagel Fat One`, 본문과 UI는 `Pretendard`를 사용한다.
- 지도, 예약, 전화 같은 외부 행동은 버튼을 명확하게 둔다.
- 여행 중 한 손으로 조작하기 쉬운 크기의 터치 영역을 사용한다.
- 장식보다 정보 밀도와 가독성을 우선한다.
- 일정 시간, 장소명, 주소, 링크는 복사하거나 열기 쉽게 만든다.
- 후보 장소 카드는 일정 화면 안에서 추천, 코멘트, 지도 열기, 상태가 한눈에 보여야 한다.

## 주요 상태

- 로딩
- 빈 목록
- 저장 성공
- 저장 실패
- 네트워크 연결 불안정
- 권한 없음 또는 잘못된 공유 링크
