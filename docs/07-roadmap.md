# 개발 로드맵

## Phase 0. 기획 정리

- 문서 구조 만들기
- 여행 기간, 도시, 멤버 확정
- 관리자 멤버 확정: 주환
- 기본 일정 입력 형식 확정
- 실제 요일별 기본 계획 정리

## Phase 1. 기본 일정 프로토타입

- 앱 기본 레이아웃
- 이름 선택 화면
- 주환 관리자 접근 코드 흐름
- 홈 화면
- 날짜별 일정 화면
- 기본 일정 데이터 표시
- 샘플 JSON 데이터 연결

## Phase 2. 후보 장소 MVP

- Supabase 연결
- 멤버 선택 저장
- 후보 장소 추가
- 구글 지도 링크 저장
- 추천 토글
- 코멘트 작성
- 방문 확인

## Phase 3. 회계 MVP

- 개인 비용과 공동 비용 입력
- EUR/KRW 입력
- 환율 자동 계산
- 공동 비용 1/N 계산
- 누가 누구에게 얼마를 보낼지 정산안 표시
- 멤버별 정산 완료 버튼
- 네 명 모두 완료 시 정산 라운드 초기화

## Phase 4. 관리자 승인

- 관리자 구분
- 승인 대기 목록
- 승인/보류/거절 처리
- 승인된 장소를 날짜/일정에 연결
- 방문 완료는 관리자만 처리

## Phase 5. 여행 전 polish

- 모바일 화면 점검
- 빈 상태와 오류 상태 정리
- 구글 지도 링크 연결
- 실제 여행 데이터 입력
- 친구들에게 공유 후 사용성 피드백 반영

## Phase 6. 여행 중 개선

- 자주 쓰는 정보 홈에 추가
- 후보 장소 필터 개선
- 방문 완료 기록 정리
- 회계 입력 흐름 단축
- 느린 네트워크에서 최소 정보 유지

## Phase 7. 나중 기능

- 체크리스트
- 중요 예약 정보
- 사진/회고

## MVP 완료 기준

- 첫 접속에서 자기 이름을 선택할 수 있다.
- 네 명이 같은 URL로 접속할 수 있다.
- 관리자가 작성한 기본 일정을 요일별로 볼 수 있다.
- 후보 장소를 구글 지도 링크와 함께 추가할 수 있다.
- 후보 장소에 추천과 코멘트를 남길 수 있다.
- 개인 비용과 공동 비용을 구분해서 기록할 수 있다.
- 공동 비용에 대해 누가 누구에게 얼마를 보낼지 볼 수 있다.
- 네 명 모두 정산 완료를 눌러야 정산 라운드가 초기화된다.
- 관리자가 후보 장소를 승인/보류/거절할 수 있다.
- 승인된 장소를 방문 완료로 표시할 수 있다.
- 모바일에서 주요 화면이 불편 없이 보인다.
