# 기본 일정 입력 템플릿

기본 일정은 앱의 첫 화면과 일정 화면에 가장 먼저 뜨는 핵심 데이터다. 이미 짜둔 여행 계획을 아래 형식으로 정리하면 바로 화면 데이터로 옮기기 쉽다.

## 작성 원칙

- 하루 단위로 나눈다.
- 각 날짜에는 요일, 도시, 핵심 이동, 주요 일정을 적는다.
- 식당/카페 후보는 기본 일정에 억지로 넣지 않고 후보 장소로 분리한다.
- 예약이 있는 항목은 시간, 예약명, 링크를 최대한 정확히 적는다.
- 시간이 미정이면 오전, 점심, 오후, 저녁처럼 느슨한 시간대로 적는다.

## 앱 데이터 초안

아래 형태로 옮기면 개발할 때 `seed` 데이터로 바로 쓰기 쉽다.

```json
{
  "trip": {
    "title": "Italy Trip",
    "startDate": "YYYY-MM-DD",
    "endDate": "YYYY-MM-DD",
    "cities": ["Rome", "Florence", "Venice"]
  },
  "members": [
    { "id": "member-1", "name": "관리자", "role": "admin" },
    { "id": "member-2", "name": "친구1", "role": "member" },
    { "id": "member-3", "name": "친구2", "role": "member" },
    { "id": "member-4", "name": "친구3", "role": "member" }
  ],
  "days": [
    {
      "id": "day-1",
      "date": "YYYY-MM-DD",
      "weekday": "요일",
      "city": "Rome",
      "summary": "도착 후 숙소 체크인, 저녁 산책",
      "items": [
        {
          "id": "item-1",
          "timeLabel": "오후",
          "title": "숙소 체크인",
          "placeName": "Hotel Name",
          "description": "짐 맡기고 주변 이동",
          "mapUrl": "https://maps.google.com/...",
          "importance": "high"
        }
      ]
    }
  ]
}
```

## 기본 일정과 후보 장소의 관계

- 기본 일정: 이미 확정된 큰 흐름
- 후보 장소: 친구들이 추가하는 식당, 카페, 관광 후보
- 승인된 후보 장소: 관리자가 기본 일정의 특정 날짜나 일정 항목에 연결

예를 들어 "Day 3 오후: 두오모 주변 자유 시간"이 기본 일정이라면, 친구들은 그 일정에 연결해서 카페 후보를 여러 개 올릴 수 있다. 관리자는 그중 하나를 승인해 실제 방문 후보로 올린다.
