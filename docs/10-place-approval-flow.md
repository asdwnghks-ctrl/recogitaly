# 장소 제안과 승인 흐름

## 목표

친구들이 부담 없이 후보 장소를 추가하고, 추천과 코멘트를 통해 의견을 모은 뒤, 관리자가 기본 동선 사이에 넣을 최종 방문지를 결정한다.

## 핵심 원칙

- 기본 여행 동선은 관리자가 짠 고정 계획이다.
- 멤버는 기본 동선을 직접 바꾸지 않는다.
- 멤버는 가고 싶은 장소의 구글 지도 링크를 복사해 후보로 추가한다.
- 후보 장소는 기본 일정과 기본 일정 사이에 연결된다.
- 관리자가 승인한 후보만 일정 화면에 확정 후보로 표시된다.
- 승인된 후보도 기본 일정 자체를 삭제하거나 대체하지 않고, 사이에 끼워 넣는 방식으로 반영한다.

## 역할

### 관리자

- 관리자는 주환이다.
- 주환은 관리자 접근 코드 입력 후 관리자 권한을 사용한다.
- 기본 일정을 작성하고 수정한다.
- 후보 장소를 승인, 보류, 거절한다.
- 승인된 장소를 기본 일정 사이의 확정 후보로 표시한다.
- 방문 완료를 처리한다.
- 필요하면 부적절한 코멘트를 삭제한다.

### 멤버

- 자기 이름을 선택해서 들어온다.
- 후보 장소를 추가한다.
- 후보 장소에 추천을 누른다.
- 코멘트를 남긴다.
- 방문 완료 상태를 확인한다.

## 멤버 표시

- 멤버는 주환, 지환, 지완, 지원 네 명이다.
- 각 멤버는 자기 아이디로 들어온다.
- 주환은 관리자 접근 코드를 입력해야 관리자 권한을 사용한다.
- 후보 장소, 추천, 코멘트, 방문 확인에는 작성자 또는 참여자 이름을 표시한다.
- 작성자 표시는 멤버별 테마 색상으로 구분한다.
- 같은 브라우저에서는 마지막으로 선택한 멤버를 저장한다.

## 상태

| 상태 | 의미 | 표시 위치 |
| --- | --- | --- |
| 제안됨 | 친구가 새로 추가한 후보 | 후보 목록, 승인 대기 |
| 승인됨 | 관리자가 방문 후보로 확정 | 일정 상세, 후보 목록 |
| 보류 | 아직 결정하지 않음 | 후보 목록 |
| 거절 | 이번 여행에서는 제외 | 후보 목록 하단 또는 숨김 |
| 방문 완료 | 실제로 방문함 | 방문 완료 목록, 일정 상세 |

## 사용자 흐름

1. 사용자가 자기 이름을 선택한다.
2. 일정 화면에서 기본 일정 박스 사이의 `이 사이에 후보 추가`를 누른다.
3. 구글 지도 링크를 붙여넣고 장소명, 도시, 카테고리, 추천 이유를 입력한다.
4. 다른 멤버들이 추천하거나 코멘트를 남긴다.
5. 관리자는 일정 화면의 승인 관리 또는 더보기의 승인 관리에서 추천 수와 코멘트를 보고 결정한다.
6. 승인하면 후보가 추가된 그 위치에 확정 후보로 표시된다.
7. 여행 중 실제 방문하면 주환이 방문 완료로 표시한다.

## 후보 장소 카드에 보여줄 정보

- 장소명
- 도시와 카테고리
- 제안자
- 제안자 테마 색상
- 추천 수
- 최근 코멘트 수
- 구글 지도 열기 버튼
- 상태 배지
- 관련 날짜 또는 일정
- 어느 기본 일정 사이에 추가됐는지

## 관리자 승인 화면에 보여줄 정보

- 승인 대기 후보
- 구글 지도 링크
- 추천한 멤버 목록
- 코멘트 요약
- 관련 날짜 선택
- 추가된 위치: 이전 기본 일정, 다음 기본 일정
- 승인, 보류, 거절 버튼
- 결정 메모

## 후보 추가 UI

일정 화면에는 점심, 디저트, 쇼핑 같은 후보 박스를 미리 여러 개 깔지 않는다. 대신 기본 일정 박스 사이마다 하나의 추가 버튼만 둔다.

```text
[기본 일정] 오전 · 포로 로마노

+ 이 사이에 후보 추가

[기본 일정] 오후 · 트레비 분수
```

버튼을 누른 뒤 카테고리를 선택한다.

- 식당
- 카페
- 디저트
- 바
- 장보기
- 관광
- 쇼핑
- 기타

이 방식은 화면을 짧게 유지하면서도, 후보가 어느 기본 일정 사이에 들어가는지 명확하게 만든다.

## MVP에서 단순화할 부분

- 회원가입은 만들지 않고 이름 선택으로 시작한다.
- 관리자 여부는 관리자 이름 또는 간단한 접근 코드로 구분한다.
- 추천은 좋아요 한 종류만 둔다.
- 코멘트는 텍스트만 지원한다.
- 장소 링크는 구글 지도 URL만 우선 지원한다.
